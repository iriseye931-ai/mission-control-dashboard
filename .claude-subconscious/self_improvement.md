# Self Improvement
_How this memory system evolves. What to update and when._

## Update Triggers
- **project_context.md**: when architecture changes, new services added, ports change, key patterns discovered
- **pending_items.md**: after every session — add TODOs, mark completed, move deferred items
- **user_preferences.md**: when Iris corrects behavior or confirms a non-obvious approach
- **session_patterns.md**: when new recurring patterns emerge
- **guidance.md**: every session — refresh with current focus and next-session suggestions

## What Makes a Good Memory
- Specific enough to be actionable (file:line, port number, exact command)
- Explains WHY not just WHAT — future Atlas needs the reasoning
- Non-obvious — if it's in the code, don't store it

## What NOT to Store
- Things obvious from reading the code
- One-off debugging steps
- Anything in CLAUDE.md already

# Session Patterns
_Recurring behaviors, what Iris typically works on, how sessions tend to go._

## Typical Session Flow
- Iris opens Claude Code, asks what's new or picks up from pending items
- Reviews repos shared from GitHub — asks "thoughts on this?"
- Builds features in mission-control-dashboard or mesh infrastructure
- Asks for plain-English explanations of technical things built
- Session ends with Iris saying thanks/awesome — Stop hook fires, worker runs

## Recurring Patterns
- GitHub repo reviews: Iris shares links, Atlas evaluates fit vs. existing stack
- "Explain this to me as someone not 100% expert" → plain English breakdown
- "Do we have this already?" → compare against existing mesh capabilities
- Evening/night sessions: long, productive, builds multiple features
- Morning sessions: shorter, reviewing what was built, asking questions

## What Works Well
- Building first, explaining after — she trusts the work
- Honest "skip it" recommendations — she appreciates not wasting time
- Connecting new tools to concrete examples from her life/workflow

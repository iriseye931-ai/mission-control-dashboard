# User Preferences
_Learned coding style, communication preferences, and patterns._

## Communication
- Direct and concise. No summaries at the end of responses
- Explain non-technical things simply, in plain English
- Address as "Punch" — not Iris
- Uppercase for emphasis is fine — she does it herself

## Code Style
- Python + TypeScript / React + FastAPI stack
- Tailwind for styling, inline style objects when precise control needed
- Zustand for React state
- No unnecessary markdown files or READMEs unless asked
- No emojis unless asked
- Prefer editing existing files over creating new ones
- No over-abstraction — don't create helpers for one-time use

## Workflow
- Local-first, everything runs on her M5 Pro Mac
- PRs for review before pushing live — never push straight to prod
- Commit often, descriptive messages
- No --no-verify, no force push to main
